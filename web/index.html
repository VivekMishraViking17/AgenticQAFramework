Test Design

Test Case Taxonomy

Every auto-generated test case carries mandatory labels for traceability and reporting.

Quality Dimensions

functional non-functional edge-case data-centric security uat accessibility api-contract regression

Metadata per Test Case

  • Priority (P1–P4)
  • Component / service label
  • Jira story link (traceability)
  • Source doc citation (Confluence/PRD)
  • Automation candidate flag (Y/N)
  • Preconditions & test data requirements
Human-in-the-Loop

Review Gates & Checklists

QA SME — Test Design Review

  • AC coverage matrix: 100% mapped
  • All TC categories per risk profile
  • Edge & negative paths present
  • Security cases for trust boundaries
  • Data setup/teardown documented
  • No duplicate or orphan TCs
  • Labels match taxonomy
  • Peer sign-off for P1 stories

SDET — Automation Review

  • Page objects / fixtures follow standards
  • Stable selectors (data-testid policy)
  • Linked to XRay test key in PR
  • CI pipeline stage defined
  • Flake mitigation (retries, explicit waits)
  • No hardcoded secrets or PII
  • Defect template pre-filled on failure

Definition of Ready

AC testable, DOD present, epic linked, component labeled, no blockers.

RACI per Gate

Clear accountable owner for intake, review, publish, codegen, triage.

Change Control

TC changes post-XRay require change ticket and re-review.

Audit Trail

Agent run ID, model version, human approver stored per artifact.

Security Policy

No PII/secrets in prompts; redact customer data in bug references.

Escalation SOP

Agent failure → manual fallback within 4 hours.
