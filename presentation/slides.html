Agentic QE Framework

End-to-End AI-Powered Quality Engineering

Cursor Agents · Jira · Confluence · XRay · Cypress/Playwright

Executive Summary

  • AI agents auto-generate labeled test cases from Jira stories (DOD + AC)
  • Context enriched from Confluence, PRDs, and customer bug history
  • QA SME review gate before XRay publication
  • Automation MCP generates Cypress/Playwright scripts
  • Defects auto-filed to Jira with full traceability

Problem Statement

  • Manual TC authoring: 4–8 hours per story
  • Inconsistent coverage — security, edge, UAT often missed
  • Knowledge siloed across Jira, Confluence, PRDs, bugs
  • Automation backlog outpaces SDET capacity
  • Weak traceability: Story → TC → Automation → Defect

Vision & Objectives

  • Accelerate: TC draft in < 15 minutes per story
  • Broaden: Functional, NFR, security, data, edge, UAT coverage
  • Preserve: Mandatory human review at every critical step
  • Trace: 100% Jira → XRay → Automation → Defect
  • Measure: KPIs on cycle time, coverage, escapes

8-Phase Lifecycle

01 Story Intake — Story Context Agent
02 Knowledge Retrieval — Knowledge Agent
03 Test Design — Design + Security + UAT Agents
04 QA SME Review — Review Orchestrator
05 XRay Sync — XRay Sync Agent
06 Automation Plan — Automation Scout
07 Script Generation — Codegen Agent
08 Execute & Defect — Execution + Triage

Phase 1–2: Intake & Knowledge

  • Story Context Agent triggered when story is Ready for QA
  • Parses description, AC, DOD, labels, component, epic
  • Knowledge Agent: RAG over Confluence, PRDs, customer bugs
  • Builds unified context bundle for test design agents
  • Human gates: PO scope + SME source validation

Phase 3: Multi-Agent Test Design

  • Test Design Agent: Functional, edge, data-centric, NFR
  • Security Test Agent: OWASP, auth boundaries, injection
  • UAT Agent: Business scenarios from PRD personas
  • Each TC: priority, component, source citation, auto flag

Test Case Taxonomy

  • functional — Core behavior per AC
  • non-functional — Performance, reliability
  • edge-case — Boundaries, empty states
  • data-centric — CRUD, validation, migration
  • security — AuthN/Z, IDOR, injection
  • uat — Business acceptance flows
  • accessibility · api-contract · regression

Phase 4: QA SME Review

  • Review Orchestrator: coverage matrix vs Acceptance Criteria
  • Checklist: 100% AC, all categories, no duplicates
  • Security cases for trust boundaries required
  • Peer sign-off for P1 stories
  • Reject → rework loop | Approve → XRay

Phase 5: XRay Publication

  • XRay Sync Agent creates/updates tests via MCP
  • Links each test to Jira story
  • Applies taxonomy labels
  • QE Lead approves publish
  • Test plan version tagged for audit

Phase 6–7: Automation

  • Automation Scout monitors XRay for approved tests
  • Prioritizes P1/P2 automation candidates
  • Codegen Agent: Cypress or Playwright scripts
  • Page objects, data-testid selectors, CI stage
  • SDET code review gate before merge

Phase 8: Execution & Defects

  • Execution Agent: CI runs, artifacts, flake detection
  • Product failures → Defect Triage Agent
  • Jira bugs: auto-filed, component, severity labels
  • Links: XRay key, screenshot, stack trace
  • Dev triage: product defect vs test fragility

MCP Integration Architecture

  • Cursor Orchestration Layer
  • Atlassian MCP — Jira, Confluence, JQL
  • XRay MCP — Test CRUD, story linking
  • Git/CI MCP — Repo, PR, pipeline
  • Test Framework MCP — Cypress/Playwright codegen

6 Human Review Gates

  • PO scope confirmation (Definition of Ready)
  • SME source validation
  • QA SME test review (mandatory checklist)
  • QE Lead XRay publish approval
  • SDET prioritization & code review
  • Dev defect triage

Measurables & KPIs

  • TC draft time: < 15 min · AC coverage: 100%
  • SME turnaround: < 1 day · XRay traceability: 100%
  • Automation (P1/P2): ≥ 70% · Codegen pass: ≥ 85%
  • Defect label accuracy: ≥ 95% · Flake rate: < 3%
  • Escape defects: trend down QoQ

Skills Matrix

  • People: QE Lead, QA SME, SDET, PO + prompt engineering
  • Process: DOR, RACI, change control, audit trail, SOP
  • Technology: Cursor, Jira, Confluence, XRay, Cypress/Playwright, MCP

Rollout Roadmap

  • Pilot (Wks 1–4): 1 squad, 10 stories, baseline
  • Integrate (Wks 5–10): XRay MCP, review automation
  • Automate (Wks 11–18): Codegen, CI, defect agent
  • Scale: All squads, prompt tuning, quarterly reviews
